Overview

A plugin is an application written in Java that runs within SQuirreL. This allows developers to enhance the functionality of SQuirreL without having to rebuild the SQuirreL application itself.

The standard plugins are included in SQuirreL's standard installation. Besides these there are stable plugins, beta plugins and plugins available at other sites. that you can download and add to your installation.

The source for each plugin is supplied with it.

  

 

Installation

To install plugins for SQuirreL SQL Client 1.1 and above, you need to download the appropriate .zip file from the Downloads page and unpack it into the plugins directory within the SQuirreL SQL Client application directory. Please be sure to preserve the directory structure when you unpack the files.

  

 

Stable Plugins

jEdit by Colin Bell and Johan Compagner
Version 0.21 released 05 March 2003

Requirements: SQuirreL SQL Client 1.1final1 and JDK1.3+. This plugin has been replaced by the Syntax Highlighting plugin for SQuirreL 1.2beta1 and above.

This plugin replaces the standard Java text area control with one from the jEdit text editor. Additional features include syntax highlighting, bracket matching and much more.

Oracle Plugin by Colin Bell
Version 0.11 released 30 September 2002

Requirements: SQuirreL SQL Client 2.0 rc1+ and JDK1.4+

This plugin adds Oracle specific functionality to SQuirreL.

Session Scripts by Colin Bell
Version 0.12 released 30 September 2002

Requirements: SQuirreL SQL Client 2.0 rc1+ and JDK1.4+

SQL Validator by Colin Bell
Version 0.12 released 30 September 2002

Requirements: SQuirreL SQL Client 2.0 rc1+ and JDK1.4+

This plugin will validate your SQL against the ISO SQL standard. It does this by connecting to a Web Service provided by Mimer SQL. The SQL that is validated is stored anonymously to be used by the ISO SQL Standards committee.

For more information on the web service click here.

Note If you use this plugin with JDK 1.3 you will need a JAXP 1.1 XML compliant parser such as xml-xerces or the JAXP 1.1 reference implementation. Copy the jar files from your XML parser to the directory <squirrel-sql>/plugins/sqlval/lib. For xml-xerces this should be xerces.jar and for JAXP 1.1 reference implementation this should be crimson.jar and jaxp.jar.

Standard Plugins

Look and Feel by Colin Bell
Version 1.0 released 21 Jun 2005

Requirements: SQuirreL SQL Client 2.0 rc1+ and JDK1.4+

This plugin allows you to select a Look and Feel other than the default Java one. It also allows you to specify the fonts for the different GUI elements. It ships with several Look and Feel implementations.

SQL Bookmarks by Joe Mocker
Version 1.0 released 21 Jun 2005

Requirements: SQuirreL SQL Client 2.0 rc1+ and JDK1.4+

This plugin allows you to manage commonly used SQL code for easy reuse. The plugin also provides a simple syntax for parameter replacement within the bookmarked SQL. As the SQL is reloaded, the user is prompted to enter values for any embedded parameters, after which the SQL is loaded into the SQL editor for execution. To select an existing bookmark use the ctrl+j shortcut or the tools popup (ctrl+t) in the SQL editor. New bookmarks can be added from the SQL editor via the tools popup (ctrl+t).

SQL Scripts by Johan Compagner and Gerd Wagner
Version 1.0 released 21 Jun 2005

Requirements: SQuirreL SQL Client 2.0 rc1+ and JDK1.4+

The SQL Scripts plugin allows you to generate CREATE TABLE and INSERT scripts for existing tables in your database. It also allows you to create data insert script from SQLs or tables. For any SELECT statement it allows you to store the result of the statement in a new table.

Syntax Highlighting Plugin by Colin Bell and Gerd Wagner
Version 1.0 released 21 Jun 2005

Requirements: SQuirreL SQL Client 2.0 rc1+ and JDK1.4+

This plugin replaces the old jEdit plugin. It's built upon the editor of Sun's Netbeans IDE editor and supplies syntax highlighting for the SQL editor. It provides search and replace functionality and it allows you to define abreviations and auto corrects in a similar way as in modern office products.

SQL Entry Completion Plugin / Code Completion by Gerd Wagner
Version 1.0 released 21 Jun 2005

Requirements: SQuirreL SQL Client 2.0 rc1+ and JDK1.4+

This plugin offers code completion in much the same way as modern IDEs do. You use code completion through the standard ctrl+space shortcut or the tools popup (ctrl+t). Completion works on tables, views, columns catalogs, schemas, aliases, SQL keywords, data types and stored procedures. It allows you to generate of SQL-JOIN syntax based on foreign keys.

SQL Entry Area Enhancements Plugin by Gerd Wagner
Version 1.0 released 21 Jun 2005

Requirements: SQuirreL SQL Client 2.0 rc1+ and JDK1.4+

This plugin provides adding and removing Java quoting around SQL and formating of SQL in the SQL editor.

Graph Plugin by Gerd Wagner
Version 1.0 released 21 Jun 2005

Requirements: SQuirreL SQL Client 2.0 rc1+ and JDK1.4+

This plugin allows you to create and print charts of database tables and their foreign key relations. If you use it together with the SQL Scripts plugin you are able to script tables from within the charts.
